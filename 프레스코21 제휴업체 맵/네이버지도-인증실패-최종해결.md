# 네이버 지도 API 인증 실패 최종 해결 방법

## 🔍 에러 분석 (스크린샷 기준)

### 발견된 문제

콘솔 로그를 보면:
```
URI: https://jiho5755-maker.github.io/-21--/
Error Code: 200 / Authentication Failed
```

**핵심 문제:** URI가 이상하게 표시됨 (`/-21--/`)

---

## ✅ 해결 방법

### 문제 1: GitHub Pages URL 등록 확인

네이버 클라우드 플랫폼 콘솔에서 Web Service URL에 다음을 **정확히** 등록해야 합니다:

```
https://jiho5755-maker.github.io
```

**중요 체크사항:**
- ✅ `https://` (프로토콜 필수)
- ✅ 슬래시(/) 끝에 없어야 함
- ✅ 포트 번호 없음
- ✅ 경로 없음 (루트 도메인만)

**잘못된 예:**
```
https://jiho5755-maker.github.io/  ← 끝에 슬래시 있으면 안 됨
http://jiho5755-maker.github.io    ← http가 아니라 https
https://jiho5755-maker.github.io/프레스코21/  ← 경로 포함하면 안 됨
```

---

### 문제 2: GitHub Pages 파일 경로 확인

GitHub Pages에서 접속하는 실제 URL 구조:
- Repository 이름에 따라 URL이 달라짐
- 예: `https://jiho5755-maker.github.io/프레스코21-제휴업체-맵/`

**해결:**
1. GitHub Repository 이름 확인
2. 실제 접속 URL 확인
3. 네이버 콘솔에 **루트 도메인만** 등록: `https://jiho5755-maker.github.io`

---

## 🔧 즉시 수정해야 할 사항

### 1. 네이버 클라우드 플랫폼 콘솔 설정

1. **콘솔 접속:** https://console.ncloud.com/
2. **Maps → Application → Pressco21-Maps** 클릭
3. **Web Service URL 섹션** 확인
4. **다음 URL이 정확히 등록되어 있는지 확인:**

```
https://jiho5755-maker.github.io
```

5. **끝에 슬래시(/) 없이** 등록
6. **저장** 버튼 클릭
7. **5~10분 대기**

---

### 2. GitHub Pages 실제 URL 확인

**Repository 구조에 따라 URL이 달라집니다:**

#### 경우 1: Repository 이름이 `프레스코21-제휴업체-맵`인 경우
- 실제 URL: `https://jiho5755-maker.github.io/프레스코21-제휴업체-맵/`
- 하지만 네이버 콘솔에는: `https://jiho5755-maker.github.io`만 등록

#### 경우 2: Repository 이름이 `jiho5755-maker.github.io`인 경우 (User/Org Pages)
- 실제 URL: `https://jiho5755-maker.github.io/`
- 네이버 콘솔에도: `https://jiho5755-maker.github.io` 등록

---

### 3. HTML 파일 확인

현재 HTML 파일에 이미 다음이 추가되어 있습니다:
```html
<meta name="referrer" content="origin">
```

이것은 올바른 설정입니다.
