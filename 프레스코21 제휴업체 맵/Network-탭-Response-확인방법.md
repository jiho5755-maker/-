# Network 탭에서 Response 확인하는 방법

## 🔍 단계별 가이드

### Step 1: 개발자 도구 열기
1. **F12** 누르기
2. 또는 우클릭 → "검사" (Inspect)

### Step 2: Network 탭 열기
1. 개발자 도구 상단 탭에서 **"네트워크"** (Network) 클릭

### Step 3: 페이지 새로고침
1. **Cmd + R** (Mac) 또는 **Ctrl + R** (Windows)
2. 또는 F5 키

### Step 4: maps.js 파일 찾기
1. Network 탭의 파일 목록에서 **"maps.js"** 파일 찾기
2. 또는 상단 검색창에 "maps" 입력하여 필터링

### Step 5: maps.js 파일 클릭
1. **maps.js** 파일을 **클릭**
2. 오른쪽 패널이 열림

### Step 6: Response 탭 확인
오른쪽 패널에 여러 탭이 있습니다:
- **Headers** (헤더)
- **Preview** (미리보기)
- **Response** (응답) ← **여기를 클릭!**
- **Timing** (타이밍)

**Response 탭**을 클릭하면 서버에서 받은 응답 내용을 볼 수 있습니다.

---

## 📊 확인할 내용

### 1. Status Code (상태 코드)
- **200** = 정상 (스크립트는 로드됨)
- **401/403** = 인증 실패
- **404** = 파일 없음

### 2. Response 내용
Response 탭에서 다음을 확인:
- HTML 응답이 있는지
- 에러 메시지가 있는지
- JSON 형태의 에러 응답이 있는지

---

## 💡 참고

네이버 지도 API의 경우:
- **maps.js 파일 자체는 200으로 로드됨** (스크립트 파일이므로)
- 하지만 **실제 API 호출(타일 로드 등)에서 인증 실패 발생**

따라서 maps.js의 Response를 확인하는 것보다, 콘솔의 에러 메시지가 더 중요합니다.
