# 네이버 지도 API 인증 실패 문제 원인 및 해결 방법

## 🔍 문제 원인 분석

### 발견된 사실

1. **GitHub Pages 실제 URL:**
   - `https://jiho5755-maker.github.io/-21--/`

2. **네이버 콘솔 등록 URL:**
   - ✅ `https://jiho5755-maker.github.io` (루트 도메인 - 올바름)
   - ❌ `https://jiho5755-maker.github.io/-21-/` (잘못된 URL)
   - ❌ `https://jiho5755-maker.github.io/` (슬래시 포함)
   - ❌ `https://jiho5755-maker.github.io/-21-` (잘못된 URL)

### 핵심 문제

네이버 지도 API는 **origin (도메인)**만 확인합니다. 따라서:
- ✅ `https://jiho5755-maker.github.io` 등록 → 하위 경로(`/-21--/`)에서도 작동해야 함
- 하지만 실제로는 작동하지 않음

**가능한 원인:**
1. 네이버 콘솔에서 **저장 버튼을 누르지 않았을 가능성**
2. 저장 후 **반영 시간 부족** (최소 10~30분 필요)
3. 잘못된 URL들이 등록되어 있어 혼선 가능

## ✅ 즉시 해결 방법

### Step 1: 네이버 콘솔에서 불필요한 URL 제거

**제거해야 할 URL들:**
- ❌ `https://jiho5755-maker.github.io/-21-/` → **삭제**
- ❌ `https://jiho5755-maker.github.io/` → **삭제** (슬래시 포함)
- ❌ `https://jiho5755-maker.github.io/-21-` → **삭제**

**유지해야 할 URL:**
- ✅ `https://jiho5755-maker.github.io` (슬래시 없이) → **유지**

### Step 2: 정확한 URL만 등록

**최종 등록 목록 (GitHub Pages용):**
```
https://jiho5755-maker.github.io
```

**로컬 테스트용 (선택사항):**
```
http://localhost
http://localhost:8080
http://127.0.0.1
http://127.0.0.1:8080
```

### Step 3: 저장 버튼 클릭 (중요!)

1. 페이지 하단의 **"✓ 저장"** (파란색 버튼) 클릭
2. "저장되었습니다" 또는 "적용되었습니다" 메시지 확인
3. **10~30분 대기** (네이버 시스템 반영 시간)

### Step 4: 브라우저 캐시 완전 삭제

1. 브라우저 완전 종료
2. 재시작
3. 시크릿 모드에서 테스트:
   ```
   https://jiho5755-maker.github.io/-21--/
   ```
4. Cmd+Shift+R (강제 새로고침)

## 💡 참고사항

### 네이버 지도 API URL 인증 방식

네이버 지도 API는 **origin (프로토콜 + 도메인 + 포트)**만 확인합니다:
- ✅ `https://jiho5755-maker.github.io` 등록
- → `https://jiho5755-maker.github.io/-21--/` ✅ 작동
- → `https://jiho5755-maker.github.io/다른경로/` ✅ 작동

하위 경로는 신경 쓰지 않으므로, **루트 도메인만 등록**하면 됩니다.

### 잘못된 URL을 등록하면?

잘못된 URL들이 등록되어 있으면:
- 네이버 시스템이 혼선을 일으킬 수 있음
- 올바른 URL도 인식하지 못할 수 있음

**해결:** 불필요한 URL들을 모두 삭제하고 올바른 URL만 등록하세요.
