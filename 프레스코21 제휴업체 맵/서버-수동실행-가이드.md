# 서버 수동 실행 가이드 (문제 해결)

## 현재 문제

1. **HTML 파일 더블클릭**: `file:///` 경로로 열림 → 네이버 지도 API 작동 안 함 (정상)
2. **서버실행.command 더블클릭**: 404 에러 발생 → 브라우저가 `localhost`만 열림

## 해결 방법

### 방법 1: 터미널에서 수동 실행 (권장)

1. **터미널 열기** (Spotlight에서 "터미널" 검색)

2. **다음 명령어 실행:**
```bash
cd "/Users/jangjiho/Desktop/커서AI/프레스코21 제휴업체 맵"
python3 -m http.server 8080
```

3. **터미널에 다음과 같이 표시되면 성공:**
```
Serving HTTP on 0.0.0.0 port 8080 (http://0.0.0.0:8080/) ...
```

4. **브라우저에서 다음 주소 접속:**
```
http://localhost:8080/제휴업체-지도-통합본.html
```

**중요:** 주소창에 정확히 위 주소를 입력하세요!

---

### 방법 2: 서버실행.command 파일 사용

1. **기존 서버 종료** (포트 충돌 방지):
   - 터미널에서: `lsof -ti:8080 | xargs kill -9`

2. **서버실행.command 파일 더블클릭**

3. **터미널 창이 열리면 서버가 실행 중인지 확인**

4. **브라우저 주소창에 직접 입력:**
   ```
   http://localhost:8080/제휴업체-지도-통합본.html
   ```

---

## 확인 체크리스트

- [ ] 서버가 실행 중 (터미널에 "Serving HTTP on..." 메시지 표시)
- [ ] 브라우저 주소창이 `http://localhost:8080`으로 시작
- [ ] 파일명이 포함됨: `제휴업체-지도-통합본.html`
- [ ] 네이버 지도가 정상적으로 표시됨
- [ ] 콘솔(F12)에 네이버 지도 API 인증 오류 없음

## 문제 해결

### 문제: 404 에러 (File not found)

**원인:** 
- 브라우저 주소창에 `localhost`만 입력됨
- 파일명이 없음

**해결:**
1. 주소창을 클릭
2. 정확한 URL 입력:
   ```
   http://localhost:8080/제휴업체-지도-통합본.html
   ```
3. Enter 키 누르기

### 문제: 포트 8080이 이미 사용 중

**원인:** 이전 서버가 아직 실행 중

**해결:**
```bash
# 기존 서버 종료
lsof -ti:8080 | xargs kill -9

# 서버 재시작
cd "/Users/jangjiho/Desktop/커서AI/프레스코21 제휴업체 맵"
python3 -m http.server 8080
```

### 문제: 서버실행.command가 브라우저를 올바르게 열지 않음

**원인:** Mac의 `open` 명령어가 URL을 제대로 파싱하지 못함

**해결:** 
- 터미널에서 수동으로 서버 실행 (방법 1)
- 또는 브라우저 주소창에 직접 URL 입력
